
## 1. Map을 스트림으로 정렬하는법


`Map.stream.sorted(Map.Entry.comparingByKey()).collect(Collectors.toMap(Map.Entry.getKey,Map.Entry.getValue,(e1, e2) → e1,LinkedList::new));`


Java의 `Collectors.toMap`은 인자 개수에 따라 사용할 수 있는 기능이 정해져 있습니다.

- **인자 2개:** `toMap(keyMapper, valueMapper)`
	- 기본 `HashMap`을 반환하며, 중복 발생 시 예외(Error)를 던집니다.
- **인자 3개:** `toMap(keyMapper, valueMapper, mergeFunction)`
	- 중복 발생 시 처리 로직(`mergeFunction`)을 추가합니다.
- **인자 4개:** `toMap(keyMapper, valueMapper, mergeFunction, mapFactory)`
	- **여기서** **`mapFactory`****가 바로** **`LinkedHashMap::new`****를 넣는 자리입니다.**

## 2. Map의 computeIfPresent

- 있으면 있는 해당 값을 반환하고, 없으면 처리해 만들어서 Map에 넣은 다음에 반환한다.
- getOrDefault와의 차이점

| **특징**         | **getOrDefault**       | **computeIfAbsent**           |
| -------------- | ---------------------- | ----------------------------- |
| **Map의 상태 변화** | **변화 없음.** 값만 빌려오는 것임. | **값이 추가됨.** Map에 실제로 데이터를 넣음. |
| **계산 시점**      | 기본값을 항상 인자로 넘겨줘야 함.    | Key가 없을 때만 람다식을 실행함.          |
| **주 용도**       | 단순히 값을 조회할 때           | 데이터가 없으면 생성해서 채워 넣어야 할 때      |
