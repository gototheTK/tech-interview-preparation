
## Q) 도커의 이미지와 컨테이너에 대해서 설명해주세요.


도커라는게 어플리케이션과 그 실행 환경을 이미지라는 파일로 저장하여, 동일한 환경에서 항상 동일한 상태를 제공하기 위해 만든 기술입니다. 자바로 비유하자면, 이미지는 클래스, 컨테이너는 불변 인스턴스라고 비유할 수 있을거같습니다. 왜 불변 인스턴스냐면 컨테이너는 한 번 띄우면 변경이 불가능합니다. 그래서 변경을 하려면 기존의 컨테이너를 내리고, 새로운 컨테이너를 띄워야합니다. 컨테이너를 흔히 격리된 프로세스라고 하는걸 들은것 같습니다. 또한, 레이어 캐싱이라는 기술을 사용하여서, 이미지가 변경되면 이미지의 해당 층만 변경합니다.
