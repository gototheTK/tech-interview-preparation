
## Q) 멀티 스레드 환경에서 발생하는 동시성 이유(Race Condition)와 자바의 해결책은?


스레드는 스택(Stack) 영역은 독립적으로 가지만, 힙(Heap) 영역은 공유합니다. 두개 이상의 스레드가 Heap의 공유 자원에 동시 접근해 읽고/수정하고/쓰는 과정에서 문맥교환이 일어나면 데이터 정합성이 깨집니다. 이에 대한 해결책으로 Java에서는 synchronized(상호 배제 락), Atomic(락 프리 원자성 보장), Concurrent 컬렉션(스레드 안전한 자료구조)등을 제공합니다.


## Q) 뮤텍스(Mutex)와 세마포어(Semaphore)의 차이는 무엇인가요?

- 뮤텍스 (상호 배제): 동기화 대상이 1개일 때 사용합니다. 락을 획득한 스레드만이 락을 해제할 수 있습니다. 즉, 락에 대한 소유권은 해당 스레드가 갖습니다.
- 세마포어(순서 제어 및 자원 관리): 동기화 대상이 N개일 때 사용합니다. 현재 락을 걸고 대기 중인 스레드가 아니더라도, 작업을 마친 다른 스레드가 Signal을 보내 락을 해제할 수 있습니다. 락에 대한 소유권은 스레드가 아닌 세마포어라는 변수가 갖습니다. 이진 세마포어(초기값 1)시 뮤텍스와의 차이점은 소유권이 없으므로 강제 해제의 위험성이 존재합니다.

## Q) 프로세스나 스레드가 교착 상태(Deadlock)에 빠지기 위한 4가지 필요조건을 설명해주세요!

- 상호배제 : 스레드가 락이 사용 중일 때는, 기다려야 한다.
- 비선점: 스레드가 락이 사용 중일 때, 뺏을 수없다.
- 점유 대기: 스레드가 락을 사용할 때, 다른 락을 요청한다
- 순환 대기: 스레드들이 사용 중인 락을 서로 요구한다.

## Q) 가상 메모리에 대해서 설명해주세요.


가상 메모리는 물리적으로 파편화되어 저장되는 프로세스의 페이지들을, 페이지 테이블을 두어서 마치 연속적으로 사용할 수 있게 해주는 기술입니다. 덕분에 프로세스는 페이티 테이블에 맵핑된 공간만을 사용할 수 있고, 다른 프로세스의 공간에 접근을 금지합니다. 만약 잘못된 포인터등으로 다른 프로세스에 접근을 시도한다면, MMU(Memory Management Unit)가 바로 감지하여서 바로 예외를 터뜨리고, 운영체자가 Segmentation Fault를 발생시켜서 해당 프로세스를 kill 시킵니다. 또한, 프로세스의 내부에서도 코드영역, 데이터영역, 힙영역, 스택영역에 따라 읽기/쓰기/실행 권한 비트를 부여하여, 코드 영역에 쓰기를 시도하는 등의 비정상적인 내부 침범도 보호합니다.
